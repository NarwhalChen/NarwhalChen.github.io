React 的工作顺序

index.html

这是应用的 HTML 模板。它包含一个根元素：


<div id="root"></div>
，React 将在此元素中挂在DOM树并渲染整个应用。

index.js

这是 React 应用的入口文件。它负责引入 React 和 ReactDOM 库，并将根组件（通常是 App 组件）挂载到 HTML 页面的根元素上。

App.js

这是应用的根组件。它可以引入和组合其他子组件。

1. 应用启动和初次渲染

  • 当你运行 npm start，Webpack 编译你的 JavaScript 代码，包括 React 组件，然后启动开发服务器。
  • 浏览器加载 index.html 文件，该文件包含一个挂载点，通常是 <div id="root"></div>。
  • index.js 被执行，其中包含对 ReactDOM.render() 的调用。这个函数负责将 React 组件（例如 App 组件）渲染到 index.html 中的指定容器（#root）内。

2. JSX 和虚拟 DOM

  • ReactDOM.render() 接收 JSX，JSX 是 React 元素的声明式描述。JSX 看起来非常像 HTML，但实际上它是通过 JavaScript 来创建虚拟 DOM 元素。
  • JSX 通过 Babel 转换成 React.createElement() 调用。这些调用生成轻量级的 JavaScript 对象，这些对象描述了组件的结构，包括它们的类型、属性、子元素等。这些对象构成了虚拟DOM。React 利用虚拟 DOM 来高效地管理 DOM 更新。在初次渲染时，React 构建一个虚拟 DOM 树，然后将这个树映射到真实的 DOM 中，这个过程称为“协调”（Reconciliation）。
  • 对于每一个 React 元素，React 会创建一个相应的 DOM 节点。这个过程包括创建 DOM 节点，并设置相应的属性和事件监听器。-
babel

(JavaScript 转译器), 将采用最新标准（ES6+）编写的 JavaScript 代码转换为向后兼容的版本，以便能在当前和旧版浏览器或环境中运行。
当 Babel 处理 JSX 代码时，具体的转换过程涉及到将 JSX 语法转换为 React.createElement() 调用。这个转换是通过 @babel/plugin-transform-react-jsx 插件（通常包含在 @babel/preset-react 中）实现的。下面详细描述了这个转换过程：
1. 解析 JSX
首先，Babel 使用其解析器将包含 JSX 的源代码转换为一个抽象语法树（AST）。在这个阶段，每个 JSX 元素（如 <div> 或 <Component />）被转换为 AST 中的 JSX 元素节点。
2. 应用 JSX 转换插件
一旦 AST 被构建，@babel/plugin-transform-react-jsx 开始工作。这个插件遍历 AST，查找所有的 JSX 元素节点，并将它们转换为 React.createElement() 的调用。
3. 生成 JavaScript 代码
一旦所有的 JSX 元素被转换为 React.createElement() 调用，Babel 的生成器（generator）将修改后的 AST 转换回普通的 JavaScript 代码。

3. 虚拟 DOM 到真实 DOM 的映射

此过程被称为“协调”（Reconciliation）

  1. JSX通过Babel变为React.createElement() 调用，生成的对象构成VDOM
  2. 比较虚拟DOM与真实DOM，通过Diff算法，React识别出需要进行更新的部分
  3. 批量更新真实DOM并更新显示
